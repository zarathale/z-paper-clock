# CODE_PROMPT — `connections.inferred[]` merge in `build_assembly_graph.py`

Extend the connection-graph extractor to read `connections.inferred[]` from each piece's JSON sidecar (`work/pieces/NNN/NNN.json`) and merge it with SVG-derived edges. Every edge in the output graph gains a `provenance` field of `"authored"` or `"inferred"`. Conflict detection between authored and inferred edges runs as a soft warning (the script never fails on conflicts).

This is the audit-side companion to DECISIONS #10 (the schema is locked there; this prompt implements the merge).

## What You Are Doing and Why

Today's connection graph (`claude-work/state/connection-graph.{json,md}`) only sees what's authored as SVG markup — `attach-X<piece>`, `landing-X<piece>`, `pivot-<name>`, etc. Knowledge from the book's instructions text and from physically assembling pieces has no structured home. With 16 panels-first pieces in the graph and the gear-train assembly ahead, that gap will widen.

DECISIONS #10 settles the schema: per-piece sidecars get a `connections.inferred[]` array carrying learned-but-not-printed cross-piece relationships, each entry mirroring the existing edge shape plus a mandatory `source` field. This prompt implements the script side — read the sidecars, merge into the graph, tag every edge with provenance.

The SVG stays the artifact-faithful authored truth; the sidecar carries learned overlay. The merger never modifies authored data based on inferred entries.

## Numbered Tasks

### 1. Add `parse_sidecar` helper

In `build_assembly_graph.py`, near the SVG-parsing helpers (around line 56–95), add:

```python
def parse_sidecar(piece_dir: Path) -> dict | None:
    """Read NNN.json sidecar if it exists. Returns parsed dict or None.

    Failures (file missing, malformed JSON) return None silently — sidecars
    are optional and the script must run cleanly with none present.
    """
    sidecar_path = piece_dir / f"{piece_dir.name}.json"
    if not sidecar_path.exists():
        return None
    try:
        return json.loads(sidecar_path.read_text(encoding="utf-8"))
    except (json.JSONDecodeError, OSError) as e:
        print(f"[build_assembly_graph] WARNING: sidecar {sidecar_path} unreadable: {e}", file=sys.stderr)
        return None
```

### 2. Add `extract_inferred_connections` helper

Add near `parse_connection_id`:

```python
def extract_inferred_connections(piece_id: str, sidecar: dict | None) -> list[dict]:
    """Extract connections.inferred[] entries from a sidecar.

    Returns a list of normalized inferred-edge records ready to merge with
    SVG-derived edges. Entries without a `kind` or `source` field are
    dropped with a warning. Partner ids are passed through as-authored
    (zero-padded or bare); the merger normalizes when matching.
    """
    if not sidecar:
        return []
    block = sidecar.get("connections", {}).get("inferred", [])
    if not isinstance(block, list):
        print(f"[build_assembly_graph] WARNING: piece {piece_id} sidecar connections.inferred is not a list", file=sys.stderr)
        return []
    out = []
    for i, entry in enumerate(block):
        if not isinstance(entry, dict):
            print(f"[build_assembly_graph] WARNING: piece {piece_id} inferred[{i}] not a dict; skipped", file=sys.stderr)
            continue
        kind = entry.get("kind")
        source = entry.get("source")
        if not kind:
            print(f"[build_assembly_graph] WARNING: piece {piece_id} inferred[{i}] missing kind; skipped", file=sys.stderr)
            continue
        if not source:
            print(f"[build_assembly_graph] WARNING: piece {piece_id} inferred[{i}] missing source; skipped", file=sys.stderr)
            continue
        normalized = {
            "from": piece_id,
            "to": entry.get("partner"),
            "kind": kind,
            "side": entry.get("side", "front"),
            "tab": entry.get("tab"),
            "letter": entry.get("letter"),
            "name": entry.get("name"),
            "panel": entry.get("panel"),
            "source": source,
            "note": entry.get("note"),
            "provenance": "inferred",
        }
        out.append(normalized)
    return out
```

### 7. Conflict detection: inferred vs. authored

After both passes complete, scan for conflicts. An inferred edge "conflicts with" an authored edge when they share `(from, to, kind, letter|tab|name|panel)`. The check is:

```python
def edge_signature(edge: dict) -> tuple:
    return (
        edge.get("from"),
        edge.get("to"),
        edge.get("kind"),
        edge.get("letter") or edge.get("tab") or edge.get("name") or edge.get("panel"),
    )

authored_sigs = {edge_signature(e) for e in edges if e.get("provenance") == "authored"}
inferred_warnings = []
for e in edges:
    if e.get("provenance") == "inferred" and edge_signature(e) in authored_sigs:
        inferred_warnings.append({
            "from": e["from"], "to": e["to"], "kind": e["kind"],
            "raw_signature": edge_signature(e),
            "inferred_source": e.get("source"),
            "message": "inferred entry duplicates an authored edge; remove the inferred entry to clean up.",
        })
```

Same check for closure-attaches and pivot members (compare on appropriate signatures).

The output graph gains a top-level `inferred_warnings: list[...]` (alongside `edges`, `pivot_clusters`, `untyped_landings`, `closure_attaches`). Markdown gets a new "Inferred conflicts" section that lists each warning. The script never fails on conflicts — it prints the count to stderr and continues.
