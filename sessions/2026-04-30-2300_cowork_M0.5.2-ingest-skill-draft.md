# Open questions / Flags

- **Empty `source/pieces/`.** The audit's first real run will report 0 captured / 123 pending. That's expected and the SKILL.md edge-cases section explicitly handles it. The cross-reference section will be useful even on the first run — it'll list which 30 pieces (across the 8 captured chunks) have a chunk source ready, so M0.5.4 cropping has a clear starting menu.
- **Threshold tuning.** WARN thresholds (DPI ≥ 600, dims ≥ 400 px, RGB/RGBA, file size ≥ 50 KB) are best-guess. They may need adjustment once gen-2 captures populate `source/pieces/` — capture findings if they surface in `work/scripts/RESCAN_FINDINGS.md` (already a Known Issue in CLAUDE.md). The skill is allowed to evolve; thresholds aren't sacred.
- **Pillow version.** SKILL.md and CODE_PROMPT both assume Pillow is in `.venv`. CLAUDE.md "Dev environment (mac)" lists it. If the Code session finds it missing, that's a `pip install Pillow --break-system-packages` away — not a blocker.
- **`--json` flag.** Defined in the contract for future programmatic consumption (e.g., a future ingest CI check), but not used by the skill itself. Authoring it now is cheap; deferring would mean a small follow-on later.
