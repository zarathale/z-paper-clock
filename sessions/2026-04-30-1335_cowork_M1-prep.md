Notable design calls baked into the CODE_PROMPT:

- **Native potrace preferred, potracer fallback** with a runtime check at startup. v2 used pure-Python potracer for sandbox compatibility; production uses native for the 50–100× speedup.
- **No downsample for production traces** — v2's `MAX_DIM = 700` was a sandbox concession; native potrace is fast enough to run full-resolution.
- **All eight canonical layers always present** in the layered SVG output, even when empty, so Inkscape's layer panel is predictable.
- **Layer-split heuristics intentionally minimal**: silhouette = largest closed path; cutouts = closed paths inside silhouette and >5% area; labels = closed paths <5% area; everything else (open paths) lands in `marks-other`. The fold-style classification (valley vs mountain), axles, and glue-zones are deferred to the Inkscape hand-edit pass (M1 task 1.5) because reliable programmatic classification from auto-trace output is brittle.
- **Validate-sidecars script is forward-looking.** Sidecars don't exist when Code's M1 session ships; the script just needs to handle empty input gracefully and exit 0. Sidecars come from a follow-up Cowork session (M1 task 1.6).
